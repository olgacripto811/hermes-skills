# Троица

Мультимодельный рабочий процесс для важных решений. Одна задача проходит через
три роли, у каждой — своя модель, играющая на своих сильных сторонах.

| Роль                | Модель (OpenRouter)          | Что делает                                              |
| ------------------- | ---------------------------- | ------------------------------------------------------- |
| 🎼 Дирижёр          | `anthropic/claude-sonnet-4-6`| Думает, планирует, ставит ТЗ, собирает финал            |
| 🔨 Работяга         | `deepseek/deepseek-v4-flash` | Делает основной объём: анализ, варианты, черновики      |
| 🔍 Критик           | `google/gemini-3.1-pro-preview` | Независимо ломает результат работяги, ищет дыры      |

Умная дорогая модель только думает и собирает итог. Дешёвая делает весь объём.
Отдельная независимая модель пытается результат сломать. Так выходит дешевле и
качественнее, чем одной моделью.

## Когда использовать

Для важных решений по стратегии, ресёрчу, контенту, нише, продукту или бизнесу,
где цена ошибки высокая.

**Триггеры:** `троица`, `используй троицу`, `прогони через троицу`,
`через 3 модели`, `council`, `red team this`, `plan-execute-critique`.

**НЕ использовать** на простых задачах: перевод, короткий ответ, поиск ссылки,
факт-чек одного утверждения. Там просто отвечай сам, без вызова трёх моделей.

## Процедура

Ты — Дирижёр. Ты НЕ делаешь основную работу сам и НЕ критикуешь сам. Твоя
задача — управлять процессом и собрать финал. Выполняй шаги строго по порядку.

### Шаг 0. Подтверди запуск

Покажи пользователю задачу одной строкой и состав ролей. Не спрашивай разрешения
повторно, если он уже явно попросил Троицу — сразу переходи к Шагу 1.

### Шаг 1. Дирижёр ставит ТЗ (роль Дирижёра — Sonnet)

> ⚠️ **Важно про модель дирижёра.** Дирижёрская работа (постановка ТЗ на этом
> шаге и сборка финала на Шаге 4) должна выполняться моделью
> `anthropic/claude-sonnet-4-6`, даже если дефолтная модель агента сейчас
> другая (например, DeepSeek). Это сохраняет смысл связки: умный дирижёр +
> дешёвый работяга + независимый критик.
>
> - Если текущая модель агента уже `claude-sonnet-4-6` — делай этот шаг и Шаг 4
>   сам, напрямую.
> - Если текущая модель другая (DeepSeek и т.п.) — выполняй дирижёрские шаги
>   через явный вызов Sonnet (см. команду ниже), а не текущей моделью.

Разбери запрос пользователя. Сформулируй для Работяги чёткое ТЗ:
- что именно нужно сделать (конкретный результат, формат, объём);
- какой контекст важен (вытащи из USER.md / MEMORY.md / истории, если есть);
- какие критерии «хорошо» (по чему будем оценивать результат).

Если дефолт НЕ Sonnet, сформулируй ТЗ через дирижёра-Sonnet:

```bash
hermes chat -q "Ты дирижёр мультимодельного процесса. Задача пользователя: \
[ЗАПРОС]. Контекст: [USER.md/MEMORY.md если есть]. Сформулируй чёткое ТЗ для \
исполнителя: что сделать, какой формат и объём, какие критерии качества. \
Только ТЗ, без выполнения." \
  --provider openrouter \
  --model anthropic/claude-sonnet-4-6 \
  -Q
```

Сохрани это ТЗ — оно понадобится на Шаге 3 и Шаге 4.

### Шаг 2. Работяга делает черновик (DeepSeek)

Вызови работягу одним one-shot запросом. Передай ему ТЗ из Шага 1 целиком.

```bash
hermes chat -q "ТЗ И КОНТЕКСТ ЗДЕСЬ. Сделай развёрнутый черновик решения." \
  --provider openrouter \
  --model deepseek/deepseek-v4-flash \
  -Q
```

Сохрани вывод как `ЧЕРНОВИК`.

### Шаг 3. Критик ломает черновик (Gemini)

Вызови критика one-shot запросом. Передай ему ТЗ + ЧЕРНОВИК. Проси НЕ хвалить,
а искать слабые места.

```bash
hermes chat -q "Вот задача: [ТЗ]. Вот черновик решения: [ЧЕРНОВИК]. \
Ты независимый критик. Найди слабые места, риски, логические дыры, \
непроверенные допущения, упущенные варианты. Не хвали. Дай список проблем \
по убыванию важности и конкретные предложения, что исправить." \
  --provider openrouter \
  --model google/gemini-3.1-pro-preview \
  -Q
```

Сохрани вывод как `КРИТИКА`.

### Шаг 4. Дирижёр собирает финал (роль Дирижёра — Sonnet)

Сведи воедино: возьми ЧЕРНОВИК, наложи на него КРИТИКУ, отбрось слабое,
усиль сильное. Сформируй финальный ответ для пользователя.

Если текущая модель агента — Sonnet, делай это сам напрямую. Если дефолт другой
(DeepSeek и т.п.) — собирай финал через явный вызов Sonnet:

```bash
hermes chat -q "Ты дирижёр. Исходное ТЗ: [ТЗ]. Черновик исполнителя: \
[ЧЕРНОВИК]. Критика независимого критика: [КРИТИКА]. Собери финальный ответ \
для пользователя: возьми сильное из черновика, учти замечания критики, \
отбрось слабое. В конце добавь блок «🔍 Что учли из критики» — 2-4 пункта." \
  --provider openrouter \
  --model anthropic/claude-sonnet-4-6 \
  -Q
```

В конце финала добавь короткий блок «🔍 Что учли из критики» — 2-4 пункта,
какие замечания критика реально повлияли на результат (чтобы пользователь
видел, что независимая проверка отработала).

## Подводные камни

- **404 / 400 от OpenRouter** — почти всегда забыт префикс вендора в slug'е
  (`anthropic/`, `deepseek/`, `google/`) или неверное имя модели. У Gemini на
  OpenRouter Pro-модели часто идут с суффиксом `-preview`
  (`google/gemini-3.1-pro-preview`), а не просто `gemini-3.1-pro`. Если slug
  устарел — проверь актуальное имя на openrouter.ai/models. Запасной стабильный
  вариант критика: `google/gemini-2.5-pro`.
- **Работяга ушёл не туда** — значит ТЗ на Шаге 1 было размытым. Уточни ТЗ и
  перезапусти Шаг 2, не правь его черновик вручную.
- **Критик хвалит вместо критики** — усиль формулировку: «твоя единственная
  задача — найти, что сломано; если всё идеально, это подозрительно».
- **Дирижёр выродился в DeepSeek** — если дефолтная модель агента не Sonnet, а
  дирижёрские шаги (1 и 4) сделаны «текущей моделью», то умного дирижёра в
  связке нет. Всегда выполняй Шаги 1 и 4 моделью `anthropic/claude-sonnet-4-6`
  явным вызовом, как описано в этих шагах.
- **Дорого** — Троица дёргает 3 модели за раз. Не запускай её на мелочах.
  Дирижёр и работяга могут работать на больших контекстах — следи за тратами.

## Проверка

Финал готов, если: (1) он отвечает на исходный ТЗ, (2) в нём виден след
критики (что-то изменилось после Шага 3), (3) нет очевидных дыр, на которые
указал критик и которые остались без ответа.
