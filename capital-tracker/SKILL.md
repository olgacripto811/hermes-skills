# capital-tracker

## Назначение
Записывает пополнения и выводы капитала инвестора в Notion базу «💰 Движение капитала».
Агент принимает текстовое сообщение → парсит данные → читает последний баланс из Notion → считает новый баланс → создаёт запись через Notion API.

## Триггеры (запускай этот скилл если пользователь пишет)
- «пополнение [сумма] от [инвестор]»
- «вывод [сумма] [инвестор]»
- «внёс [сумма]», «вывел [сумма]»
- «запиши транзакцию», «новая транзакция»
- «инвестор пополнил», «инвестор вывел»
- любое сообщение о движении капитала с суммой в USDT

## Переменные окружения (из ~/.hermes/.env)
- NOTION_API_KEY — ключ интеграции Notion
- CAPITAL_DB_ID=389a02a5-cdf9-4291-8b8a-a4f0b9ff5979

## Алгоритм выполнения

### Шаг 1 — Парсинг входящего сообщения
Извлеки из текста пользователя:
- **тип**: «Пополнение» или «Вывод»
- **сумма**: число в USDT (всегда положительное)
- **инвестор**: имя (если не указано — оставь пустым)
- **дата**: если указана — используй, иначе сегодняшняя дата в формате YYYY-MM-DD
- **заметка**: любой дополнительный текст

### Шаг 2 — Получить последний баланс из Notion
Выполни curl для получения последней записи (сортировка по дате убыванию, limit=1):

```bash
source ~/.hermes/.env

LAST=$(curl -s -X POST "https://api.notion.com/v1/databases/${CAPITAL_DB_ID}/query" \
  -H "Authorization: Bearer $NOTION_API_KEY" \
  -H "Notion-Version: 2022-06-28" \
  -H "Content-Type: application/json" \
  -d '{
    "sorts": [{"property": "Дата", "direction": "descending"}],
    "page_size": 1
  }')

PREV_BALANCE=$(echo "$LAST" | python3 -c "
import sys, json
data = json.load(sys.stdin)
results = data.get('results', [])
if results:
    props = results[0].get('properties', {})
    bal = props.get('Баланс после', {}).get('number')
    print(bal if bal is not None else 0)
else:
    print(0)
")
```

### Шаг 3 — Рассчитать новый баланс
```bash
NEW_BALANCE=$(python3 -c "
prev = float('${PREV_BALANCE}')
amount = float('СУММА_ИЗ_ШАГА_1')
ttype = 'ТИП_ИЗ_ШАГА_1'
if ttype == 'Пополнение':
    print(prev + amount)
else:
    print(prev - amount)
")
```

### Шаг 4 — Записать в Notion
```bash
TITLE="ТИП_ИЗ_ШАГА_1 ДАТА_ИЗ_ШАГА_1"
INVESTOR="ИМЯ_ИЗ_ШАГА_1"
AMOUNT=СУММА_ИЗ_ШАГА_1
DATE="ДАТА_ИЗ_ШАГА_1"
NOTE="ЗАМЕТКА_ИЗ_ШАГА_1"

curl -s -X POST "https://api.notion.com/v1/pages" \
  -H "Authorization: Bearer $NOTION_API_KEY" \
  -H "Notion-Version: 2022-06-28" \
  -H "Content-Type: application/json" \
  -d "{
    \"parent\": {\"database_id\": \"${CAPITAL_DB_ID}\"},
    \"properties\": {
      \"Название\": {
        \"title\": [{\"text\": {\"content\": \"${TITLE}\"}}]
      },
      \"Дата\": {
        \"date\": {\"start\": \"${DATE}\"}
      },
      \"Тип\": {
        \"select\": {\"name\": \"ТИП_ИЗ_ШАГА_1\"}
      },
      \"Сумма USDT\": {
        \"number\": ${AMOUNT}
      },
      \"Инвестор\": {
        \"rich_text\": [{\"text\": {\"content\": \"${INVESTOR}\"}}]
      },
      \"Баланс после\": {
        \"number\": ${NEW_BALANCE}
      },
      \"Заметка\": {
        \"rich_text\": [{\"text\": {\"content\": \"${NOTE}\"}}]
      }
    }
  }"
```

### Шаг 5 — Подтвердить пользователю
После успешной записи сообщи:

```
✅ Записано в Движение капитала

Тип:           Пополнение / Вывод
Сумма:         +500 USDT
Инвестор:      Иван
Дата:          27.06.2026
Баланс после:  1 500 USDT
```

Если ошибка — покажи HTTP-ответ Notion и предложи проверить CAPITAL_DB_ID и NOTION_API_KEY.

## Правила
- Сумма в записи ВСЕГДА положительная (знак определяет поле «Тип»)
- Если инвестор не назван — поле «Инвестор» оставь пустым, не выдумывай имя
- Если дата не указана — используй сегодняшнюю (date +%Y-%m-%d)
- Название формируй как «Пополнение 27.06.2026» (дата в формате DD.MM.YYYY для читаемости)
- Баланс не может быть отрицательным — предупреди пользователя если вывод превышает текущий баланс
- Не спрашивай «куда записать» — всегда пиши в CAPITAL_DB_ID
